# Security Policy

## Supported Versions

Security-sensitive issues should be reported for the latest public release.

## Reporting a Vulnerability

If you believe you found a security issue in **Navi Overlay**:

1. Do **not** publish exploit details immediately.
2. Open a private channel with the maintainer if possible.
3. Provide:
   - affected version;
   - reproduction steps;
   - expected impact;
   - screenshots or logs if relevant.

## Scope

Relevant reports may include issues related to:

- permission handling;
- unintended overlay behavior;
- notification data handling;
- accessibility or usage-access misuse;
- privacy-sensitive behavior.

General UX bugs and compatibility issues should be reported through standard GitHub issues.
